What GenomeHub Does

The network, not the cloud.

01 / Distributed Seeding

Any node can host any assembly.

Coverage grows with the network. No central servers. A genome seeded by one node is available to all — as long as at least one peer holds it.

● 4 MB segmentsSHA-256 verified
● Content-addresseddedup guaranteed
● Partial seedingresume any time
02 / Verified Integrity

Every segment cryptographically signed.

Reference assemblies are too critical to trust without verification. GenomeHub validates every segment on receipt. Corruption and tampering are refused automatically.

● Ed25519 signaturesper-segment
● Merkle manifestper-assembly
● NCBI provenancecross-checked
03 / Open Protocol

BSD-licensed. Documented spec.

Anyone can build compatible software. The wire format is documented, the tracker API is public, and the reference implementation is open source. Science doesn't lock in.

● BSD-2-Clauselicense
● REST tracker APIOpenAPI 3.1
● Written in Gosingle binary
How It Works

Four steps to seeding.

01
▶
Import

Add assemblies from NCBI, EBI, or local FASTA/2bit. The node fetches, validates, and segments them automatically.

$ genomehub add GRCh38.p14
02
▶
Segment

Files are split into 4 MB segments, each indexed by SHA-256. A Merkle manifest is signed and registered with the tracker.

✓ 50,000 segs · 195 GB
03
▶
Serve

Your node responds to peer requests over HTTP/2. Segments are served directly; the tracker is only consulted for discovery.

GET /seg/GRCh38.p14/chr3/seg_14219
04
Sync

The tracker coordinates availability. Nodes discover peers, fetch missing segments, and propagate delta updates automatically.

Get Started

Run a node in minutes.

A single binary, no dependencies. Bring your own storage — a few terabytes is enough to seed the most common reference assemblies.

Terminal bash
# Install
$ curl -sSL get.genomehub.org | sh
# Initialize a node
$ genomehub init --data-dir /mnt/genomes --port 7741
# Start seeding
$ genomehub node start
# Add your first assembly
$ genomehub add GRCh38.p14
